Production Roadmap

From MVP to FedRAMP High compliant, multi-tenant platform.

1

Multi-Tenant Secret Isolation

Namespace-per-customer architecture with dedicated IAM roles and path-restricted policies.

‚ñ∏ Separate IAM roles per customer: customer-a/* ‚Üí Role A, customer-b/* ‚Üí Role B
‚ñ∏ ExternalSecret per namespace, ClusterSecretStore referencing correct SA
‚ñ∏ RBAC prevents cross-namespace ServiceAccount token projection
2

RBAC & Network Hardening

Eliminate cluster-admin, enforce least privilege, isolate customer workloads.

‚ñ∏ Granular RBAC per ServiceAccount (no wildcards, explicit resource names)
‚ñ∏ NetworkPolicies: deny-all default, explicit allow rules per customer namespace
‚ñ∏ Job pods run unprivileged, no hostPath mounts, readOnlyRootFilesystem
3

Policy Enforcement (OPA Gatekeeper)

Admission control ensuring all deployments meet security baselines.

‚ñ∏ Block privileged pods, enforce Pod Security Standards (restricted)
‚ñ∏ Require resource limits/requests on all containers
‚ñ∏ Deny hostNetwork, hostPID, hostIPC usage
4

Ephemeral Runners

On-demand job pods with autoscaling, cost optimization, and blast radius reduction.

‚ñ∏ Runner manager stays up, job pods spawn/terminate per job
‚ñ∏ Fresh environment per run, no state contamination
‚ñ∏ HPA-based scaling tied to GitLab queue depth
5

Observability & Audit

Centralized logging, metrics, and alerting for FedRAMP compliance.

‚ñ∏ EKS control plane + pod logs ‚Üí CloudWatch/Splunk integration
‚ñ∏ Prometheus + Grafana dashboards: ESO sync status, IRSA auth failures
‚ñ∏ PagerDuty alerts on secret sync failures, IAM permission denials
6

Secret Rotation & GitOps

Automated secret lifecycle management with infrastructure as code.

‚ñ∏ Secrets Manager auto-rotation policies per customer
‚ñ∏ ESO manifests in Git (ArgoCD/Flux), drift detection
‚ñ∏ Terraform/Pulumi for IAM roles, OIDC, Secrets Manager setup
7

FedRAMP High Hardening

Final compliance controls for authorization boundary and continuous monitoring.

‚ñ∏ Pod Security Admission (restricted), image scanning (Trivy/Anchore)
‚ñ∏ Signed images only (Cosign/Notary), immutable infrastructure (no SSH to nodes)
‚ñ∏ SIEM integration (Splunk), KMS key rotation for Secrets Manager/EBS
